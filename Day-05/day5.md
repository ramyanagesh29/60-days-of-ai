# 🚀 Day 5 – Context Engineering

## Prompt A (Without Context)

Generated a generic 30-day Java learning roadmap.

### Observation

The roadmap was useful but assumed I was a complete beginner. It did not consider my current skills, career goal, available time, or preferred learning style.

---

## Prompt B (With Context)

Context Provided

- Current Situation: Student
- Current Skills:
  - Java (Basic)
  - Python (Basic)
  - HTML
  - CSS
  - C Programming
- Goal:
  Become a Full Stack Developer
- Available Time:
  1 Hour per Day
- Experience Level:
  Beginner
- Preferred Learning Style:
  Projects + Videos

---

## Comparison

### Which roadmap feels more personalized?

The roadmap generated using context felt much more personalized because it matched my current knowledge and focused on becoming a Full Stack Developer.

### Which roadmap would I actually follow?

The roadmap with context because it aligns with my career goal and builds on my existing skills.

### What role did context play?

Context helped Claude understand my background, learning preferences, available time, and career goal, resulting in a more practical and realistic learning plan.

---

## Key Learnings

- Better context leads to better AI outputs.
- Personalized prompts produce more relevant recommendations.
- Context reduces assumptions and improves accuracy.
